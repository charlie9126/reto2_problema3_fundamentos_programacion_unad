☕ Registro de Ventas Diarias - Tienda de Café
Este proyecto es una solución práctica desarrollada para el curso de Fundamentos de Programación, bajo la tutoría de Javier Mauricio López Moreno. El objetivo es automatizar el seguimiento financiero diario de una tienda de café.

📋 Descripción del Problema
El administrador de la tienda requiere registrar las ventas en tres momentos clave del día: Mañana, Tarde y Noche. El sistema debe procesar estos datos para determinar si se cumplieron los objetivos comerciales de la jornada.

Requerimientos del Programa:
Captura de datos: Solicitar al usuario el valor de las ventas en los tres turnos.

Procesamiento: Calcular la sumatoria total de las ventas.

Evaluación: Comparar el resultado frente a una meta diaria de $150.000.

🛠️ Tecnologías Utilizadas
Lenguaje: Python 3.11.9

Conceptos aplicados: Variables, constantes, entrada/salida de datos y estructuras de control (if-else).

🚀 Lógica de Control
El programa opera bajo la siguiente estructura lógica para determinar el éxito de la jornada:

Condición,Salida Esperada
Total >= 150.000,"""Meta alcanzada"""
Total < 150.000,"""Meta no alcanzada"""

📂 Estructura del Ejercicio
Fase: Fase 2 - Variables, constantes y estructuras de control.

Banco de problemas: Anexo - Problema #3.

Ejemplo de Uso:
Ingrese ventas de la mañana: 50000
Ingrese ventas de la tarde: 60000
Ingrese ventas de la noche: 45000

Total vendido: $155.000
Resultado: Meta alcanzada 🎉
